# 功能设计文档

本文档收录 TrailSnap 待开发功能的设计草案，供社区讨论和贡献参考。

---

## 1. 图片编辑功能

### 1.1 背景与目标

用户当前只能查看照片，缺少基本的编辑能力。图片编辑是相册应用的高频需求，可为后续"自动生成朋友圈九宫格"等功能打下基础。

### 1.2 功能范围

**基础编辑**
- 裁剪（自由裁剪、比例裁剪如 1:1、16:9）
- 旋转（90°、水平翻转、垂直翻转）
- 缩放与拖拽

**标签与标注**
- 添加文字标签（支持字体、颜色、大小）
- 添加箭头、框选等简单图形
- 添加时间、地点水印

**滤镜与调整**
- 基础调整：亮度、对比度、饱和度
- 预设滤镜：复古、黑白、柔和等

### 1.3 技术方案

- 前端使用 Canvas API 或 WebGL 实现无损编辑
- 编辑结果支持导出为原图质量的 JPEG/PNG
- 编辑历史记录（Undo/Redo）

### 1.4 优先级

**中优先级** - 作为工具箱模块的一部分开发

## 2. 照片自动整理

### 2.1 背景与目标

用户照片随时间积累杂乱，手动整理耗时。通过 AI 自动按时间、地点、内容等维度归类，减少用户操作成本。

### 2.2 整理规则

| 规则 | 说明 | 示例 |
| --- | --- | --- |
| 按年份月份 | 创建 `2024/2024-03` 文件夹 | 2024/2024-03 |
| 按日期 | 创建 `2024/2024-03-15` 文件夹 | 2024/2024-03-15 |
| 按地点 | 创建 `2024-北京/` 文件夹 | 北京故宫照片 |
| 按人物 | 创建 `人物-张三/` 文件夹 | 张三的人脸照片 |
| 按场景 | 创建 `美食/`、`宠物/` 文件夹 | 分类后的照片 |

### 2.3 技术方案

- 使用现有元数据（EXIF、地理位置、人脸标签、场景分类）
- 复制或移动照片（用户选择）
- 支持预览整理结果后再执行
- 记录整理日志，支持撤销

### 2.4 优先级

**中优先级**

## 3. 自动旅行日志

### 3.1 背景与目标

用户在旅行后常需手动整理照片、编写游记，过程繁琐。AI 可自动从照片、票据、地点等数据生成结构化旅行记录。

### 3.2 输入数据

- 旅行期间的照片（含元数据）
- 识别出的火车票、飞机票、门票
- 地理位置轨迹
- 可能的 AI 描述与评分

### 3.3 输出内容

**结构化日志**
- 标题：旅行名称（如"2024年3月厦门之旅"）
- 时间：出发日期、返回日期、总天数
- 行程：每日行程时间线
- 地点：去过的城市/景点
- 人物：同行者
- 费用：交通、门票等分类统计

**图文混排**
- 支持导出为 Markdown/HTML
- 可直接发布到博客平台
- 支持生成 PDF

### 3.4 技术方案

- 使用 LangChain/LangGraph 编排多步骤生成流程
- 分阶段生成：行程概览 → 每日详情 → 费用统计 → 最终润色
- 支持流式输出，展示生成进度
- 提供模板引擎，支持自定义输出格式

### 3.5 优先级

**高优先级** - 核心差异化功能

## 4. 朋友圈九宫格生成

### 4.1 背景与目标

用户分享旅行照片到朋友圈时，需手动拼图、选图、配文。提供一键生成九宫格及配套文案，降低分享成本。

### 4.2 功能模块

**智能选图**
- 从旅行照片中 AI 选出 9 张最具代表性的照片
- 考虑因素：质量评分、内容多样性、时间分布

**拼图样式**
- 3×3 标准九宫格
- 自由排版（支持 1+2+6 等异形布局）
- 样式模板：简约、文艺、复古

**文案生成**
- 根据照片内容、景点生成文案
- 支持多种风格：诗意、幽默、简短
- 可选择添加话题标签

### 4.3 技术方案

- 使用 Canvas API 实现拼图
- 调用大模型生成文案（few-shot prompts）
- 支持预览和微调

### 4.4 优先级

**高优先级** - 强社交传播属性

## 5. 视频高光剪辑

### 5.1 背景与目标

静态照片难以传达旅行氛围。从 Live Photo、视频素材中自动剪辑 15-30 秒的高光短片，适合分享到短视频平台。

### 5.2 功能模块

**素材收集**
- 支持 Live Photo 视频片段
- 支持用户上传的视频文件
- 支持按相册/时间筛选素材

**智能剪辑**
- 自动识别高光时刻（如笑容、动作、风景转换）
- 自动匹配合适的背景音乐
- 生成多种时长版本（15s/30s/60s）

**后期处理**
- 转场效果（淡入淡出、滑动）
- 基础调色
- 添加文字标题/字幕

### 5.3 技术方案

- 使用 OpenCV 进行视频分析（场景检测、光学流）
- 使用 FFmpeg 进行视频处理与导出
- 或接入云服务（如剪映 API）

### 5.4 优先级

**低优先级** - 技术复杂度高，可后期考虑

## 6. MCP 协议支持

### 6.1 背景与目标

AI Agent 生态正在形成，支持 MCP（Model Communication Protocol）协议可使 TrailSnap 作为本地知识库被多种 AI 平台调用。

### 6.2 协议设计

**Server 实现**
- 实现 MCP Server，提供标准化的工具调用接口
- 工具列表：搜索照片、查询相册、获取年度统计等

**认证与安全**
- API Token 验证
- 权限分级（只读、读写）

### 6.3 工具定义

```
tools:
  - name: search_photos
    description: 按关键词搜索照片
    input: { query: string, limit?: number }
    output: Photo[]

  - name: get_album_by_person
    description: 获取某人的所有照片
    input: { person_name: string }
    output: Photo[]

  - name: get_annual_report
    description: 获取年度报告数据
    input: { year: number }
    output: AnnualReport

  - name: get_travel_timeline
    description: 获取旅行时间线
    input: { start_date: string, end_date: string }
    output: TravelEvent[]
```

### 6.4 优先级

**中优先级** - 面向开发者的高级功能

## 7. 多端同步与备份

### 7.1 背景与目标

用户可能在多个设备使用 TrailSnap，需要照片库同步机制。同时提供可靠的备份方案，防止数据丢失。

### 7.2 功能模块

**增量同步**
- 检测新增、删除、修改的照片
- 仅传输差异（哈希比对）
- 支持 LAN 内网高速同步

**备份机制**
- 支持备份到外部磁盘/NAS
- 压缩包备份（ZIP/TAR）
- 增量备份（只备份变更部分）
- 备份校验（SHA256）

**恢复功能**
- 支持从备份恢复完整数据
- 支持选择性恢复（只恢复某时间段）

### 7.3 技术方案

- 使用 Rsync 协议或自定义增量同步算法
- 数据库备份使用 pg_dump
- 文件校验使用 SHA256

### 7.4 优先级

**中优先级** - 数据安全是刚需

## 8. 家庭共享与协作

### 8.1 背景与目标

家庭成员可能共用一个 TrailSnap 实例，或希望共享特定相册给家人。支持多用户协作场景。

### 8.2 功能模块

**权限管理**
- 管理员、普通用户角色
- 相册级权限控制（查看、编辑、分享）
- 访客模式（仅查看指定相册）

**相册分享**
- 生成分享链接（可选密码保护）
- 有效期控制（24小时/7天/永久）
- 禁止下载开关

**评论与点赞**
- 家庭成员可对照片评论
- 简单点赞反应

### 8.3 技术方案

- 复用现有用户系统，增加角色字段
- 分享链接使用 JWT 或临时 Token
- 评论存储在独立表

### 8.4 优先级

**低优先级** - 涉及多用户场景可后续考虑

## 9. 高级语义搜索

### 9.1 背景与目标

当前搜索基于 EXIF 元数据和人脸/OCR 结构化数据。引入多模态 Embedding 实现语义搜索，可按自然语言描述找到照片。

### 9.2 示例

| 自然语言查询 | 对应SQL/语义 |
| --- | --- |
| "去年夏天在海边的照片" | time: 去年夏季 + location: 海边 + scene: 风景 |
| "和爸妈一起吃的火锅" | person: 爸妈 + scene: 美食 + object: 火锅 |
| "国庆节拍的照片" | time: 10月1日-7日 + year: 去年 |

### 9.3 技术方案

- 使用多模态模型（如 CLIP）提取图片向量
- 存储向量到 PostgreSQL + pgvector
- 查询时将自然语言转为向量，检索最近邻

### 9.4 优先级

**高优先级** - 大幅提升搜索体验

## 10. 智能相册推荐

### 10.1 背景与目标

用户可能不会主动创建相册。系统可主动推荐"值得整理"的相册，如"周末小聚"、"十一长假"等。

### 10.2 推荐场景

| 场景 | 触发条件 |
| --- | --- |
| 周末小聚 | 周六/日有 5+ 张照片，集中在一地 |
| 十一长假 | 国庆假期有 20+ 张照片 |
| 美食集锦 | 近期"美食"分类有 10+ 张照片 |
| 家人时刻 | 近期含"父母""子女"标签照片居多 |
| 景点打卡 | 某 5A 景区有 3+ 张照片 |

### 10.3 技术方案

- 定时任务扫描，分析近期照片分布
- 基于规则+AI 识别生成推荐
- 展示给用户确认后创建相册

### 10.4 优先级

**中优先级** - 提升用户活跃度
