## Agent 对话系统设计

### 功能描述

现在的对话聊天记录是存储在内存中的，并且不支持多会话，当用户关闭应用或者重启应用时，聊天记录会丢失。
为了避免聊天记录丢失，需要将聊天记录存储在本地数据库中。

### 数据库设计
下面的表描述仅供参考，你可以根据需要修改或者增加新的字段以适应你的需求，或者未来留作功能扩展。

1. 会话表（agent_sessions）
   1. id: 会话ID，主键(UUID)
   2. user_id: 用户ID，外键，引用用户表的user_id
   3. created_at: 创建时间，记录会话创建时间
   4. title: 会话标题，用户可以自定义会话标题
   5. status: 会话状态，记录会话是否已结束
   6. context_summary: 会话上下文摘要，记录会话的上下文摘要
   7. summary_update_time: 会话上下文摘要更新时间，记录会话上下文摘要最后更新时间

2. 消息表
   1. id: 消息ID，主键(自增)
   2. session_id: 会话ID，外键，引用会话表的id
   3. role: 消息角色，角色：user/assistant/system
   4. content: 消息内容，记录用户发送的消息或Agent回复的消息
   5. content_type: 消息内容类型，记录消息内容的类型，例如文本、图片、音频等
   6. content_ext: 消息内容扩展，记录消息内容的扩展信息，例如图片的URL、音频的URL等
   7. token_count: 消息令牌数量，记录消息内容的令牌数量，用于计费
   8. created_at: 创建时间，记录消息发送时间

## 足迹时间轴功能

## 一、功能整体定义

### 1. 功能名称
相册足迹时间轴

### 2. 功能定位
基于照片**拍摄时间+省/市/区/景区/详细地址**多级地理位置，自动生成可视化时间+地理轨迹，支持多视图浏览、筛选、编辑，满足用户查看照片生活/旅行足迹的需求。

### 3. 适用终端
PC端（网页/客户端）、移动端（手机/平板），双端功能一致，展示布局自适应。

---

## 二、基础展示规则
### 1. 排序规则
1. 所有照片**按拍摄时间倒序**展示（最新时间在最上方）；
2. 同一时间、同一地理位置的照片自动聚合为一组；
3. 无地理位置的照片，不考虑时间轴展示。

### 2. 地理位置展示优先级（固定规则）
功能按以下优先级展示地点名称，**自动匹配，不可更改优先级**：
1. 第一优先级：**景区**（有景区信息时，展示：城市+景区）
2. 第二优先级：**区县**（无景区、有区县时，展示：城市+区县）
3. 第三优先级：**城市**（无景区/区县时，仅展示城市）
4. 第四优先级：**省份/国家**（无城市时，展示省份/国家）

### 3. 分组聚合规则
1. **按天分组**：同一天拍摄的照片，默认合并为一个时间节点；
2. **按地点分组**：同一天、同一地点（景区/区县/城市）的照片，合并为一个足迹点；
3. **连续行程聚合**：连续3天及以上在同一城市拍摄，自动聚合为「XX城市·N天行程」，可展开查看每日明细。

## 四、多视图展示功能（双端通用）
提供**2种独立视图**，用户可自由切换，视图数据完全同步。

### 视图1：标准足迹时间轴（默认视图）
1. 垂直时间轴布局，左侧展示**时间+地理位置**，右侧展示对应照片封面缩略图，点击可以查看该分组所有照片；
2. 单条足迹展示格式：
   - 有景区：`2026-03-25 · 杭州市·西湖景区`
   - 有区县：`2026-03-24 · 北京市·朝阳区`
   - 仅城市：`2026-03-23 · 上海市`
3. 折叠/展开：支持折叠历史月份/年份，快速收缩视图。

### 视图2：地图轨迹视图
1. 左侧：时间轴缩略列表；右侧：地图面板；
2. 地图自动标记所有照片拍摄地点，**用连线生成行走轨迹**；
3. 地图标记点规则：
   - 标记点大小：对应地点照片数量越多，标记点越大；
   - 点击标记点：自动展示该地点所有照片+时间+地址；
4. 支持轨迹隐藏/显示、地图全屏查看。

2. 时间轴不一定是按月分组，如果多个月都在一个城市，那么这个城市的所有照片都放在一个时间节点中
3. 行程不一定是连续的日期，即使某一天没有照片，但是接下来的照片还是在同一个地方则也分到同一组，关键看当前位置和下一个位置是否相同，只要位置发生了变化，就会分为不同的组

功能优化：

   1. 采用列表存储所有时间轴的item项
