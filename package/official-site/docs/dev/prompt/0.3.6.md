## 智能体对话加入思考过程和工具调用可视化

现在的对话只显示最终结果，等待时间比较长，没有中间过程，需要加入思考过程和工具调用可视化，提高用户体验。

package\server\app\db\models\agent.py agent_messages表新增一些字段，用于存储思考过程和工具调用信息。

- reasoning: 思考文本，用于存储智能体的思考过程
- tool_calls: json格式的工具调用信息，用于存储智能体调用的工具列表，每个工具调用信息包含以下字段：
  - tool_name: 工具名称
  - tool_args: 工具参数
  - tool_return: 工具返回结果
  - tool_call_id: 工具调用ID，用于唯一标识每个工具调用
  - tool_status: 工具状态，用于存储工具调用的状态，如成功、失败等

package\server\app\service\agent\service.py 将思考内容和工具调用信息存储到数据库中。

package\website\src\views\agent\AgentChat.vue 加入思考过程可视化，思考内容用浅灰色显示，可以折叠和展开，思考的时候展开，思考完成之后折叠。工具调用先不给前端展示。

## 后台任务管理优化

现在的任务管理进程是有main.py启动，一段时间没有任务会自动重启，现在需要修改为有任务时才启动，没有任务时关闭。

需要package\server\app\service\task_manager.py中新增后台任务进程管理的功能，包括启动、关闭、重启等操作。

当用户发起任务时（扫描文件夹、重试任务等），需要启动后台任务进程，任务完成一段时间后自动关闭进程，任务进程不再常驻后台。
